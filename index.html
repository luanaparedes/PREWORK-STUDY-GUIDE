CSS

  • Three ways to style webpage: Inline CSS, Internal CSS Style Sheet and External CSS Style Sheet
  • Inline: apply to specific HTML tag using a style attribute with a CSS rule to style a particular page element only, less flexible than Internal and External because each inline style must be edited individually.
  • Internal: contains CSS rules for the webpage in the head section of the HTML file, rules only apply to that page any changes to CSS rule will apply to all tagged page elements
  • External: allows developers to keep all their CSS rules in a specific file using link element placed in HTML head
  • Separation of Concerns: refers to separationg code into different files based o use
  • Declarations: contains CSS property we want to apply and the value of the property
  • Margin property: acts like margin in word doc, indicates how much space we want around outside of an element
  • Padding property: adds space arount the content inside an element, increase padding makes element bigger
  • CSS Box Model: every HTML is essentially a box, each box consists of margins outside the element and the elements borders, padding and content
  • box-shadow property: adds a shadow effect around eleents frame

Git

  • git: command must precede every git statement
  • git status: checks current branch
  • git checkout -b branch name: creates new branch and switches to it
  • feature/...: indicates it is a feature before name of new branch
  • -a flag indicated we want to add all changes to next commit
  • add: adds modification in current working branch to staging area
  • pull: recieve a branchs modification into local enviornment
  • origin: indicates source of pull will be from Github repo
  • push: creates new remote branch and adds new changes to existing branch

JavaScript

  • Variable: a named container that allows us to store data in our code
  • var: declare variable in JavaScript
  • each line of JavaScript code ends in ;"
  • Control Flow: the order in which a computer executes code in a script
  • if statement: a way of introducing decision-making into our code
  • Array: single variable that is used to hold a group of data
  • Index: location of data item in an array is identified with a unique number
  • for loop: uses predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over
  • Function: core component of many programming languages, is a set of instructions that tells the computer how to perform a certain task
