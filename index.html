HTML

  • Head element contains metadata about webpage not visible to user
  • Body contains conntent that is shown to user
  • Attribute defines additional information about ana element, always defined in beginning of element
  • Style element defines CSS style
  • Title element defines title that shows up in SEO
  • Base elenent defines base URL for the system
  • Link element connects external resources to HTML resources
  • Meta element defines metadata
  • Script element embeds client*side scripts in HTML document
  • Charset element that sets character encoding
  • http-equiv element sets an HTTP header for content
  • Viewport element sets default view for webpage
  • Body element content seen by users
  • Div creates division on page
  • Span creates inline division
  • H1 - H6 elements represent level of heading a given text block represents
  • P paragraph or block of text
  • UL OL LI elements represent unordered lists, ordered lists and list items, bulleted or numbered lists
  • IMG element contains info about image displayed on webpage
  • SRC attribute defines location of image
  • BR element creates a line of empty space between two blocks of content
  • A or anchor element creates links to webpages
  • HREF attribute points to URL for link
  • Footer appears at the vottom of page contains author,copyright, contacts, etc.

Git

  • git: command must precede every git statement
  • git status: checks current branch
  • git checkout -b branch name: creates new branch and switches to it
  • feature/...: indicates it is a feature before name of new branch
  • -a flag indicated we want to add all changes to next commit
  • add: adds modification in current working branch to staging area
  • pull: recieve a branchs modification into local enviornment
  • origin: indicates source of pull will be from Github repo
  • push: creates new remote branch and adds new changes to existing branch
